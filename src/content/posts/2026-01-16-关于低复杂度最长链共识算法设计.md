# 1.基于同步网络假设

算法假设网络是同步的，即消息传递延迟有上界，节点时钟同步。首先，建立了基于网络动力学的最长链收敛模型，通过节点动力学方程描述区块传播和链收敛过程。其次，设计了基于本地决策的最长链选择策略，节点仅需与部分邻居通信即可确定最长链，将通信复杂度从 O(N^2)降低到 O(N\log N)。最后，设计了基于链权重的快速收敛机制，通过赋予不同区块权重加速最长链收敛。

同步网络假设是指网络消息传递延迟有上界，节点时钟同步。在同步网络假设下，可以设计更高效的共识算法。

### 2.1 同步网络模型

**网络模型**：

考虑一个区块链共识网络，用无向图 G = (V, E)表示，其中 V = \{1, 2, \ldots, N\}表示共识节点集合，E \subseteq V \times V 表示节点间的连接集合。

**同步网络假设**：

1. **消息传递延迟有上界**：存在常数\Delta > 0，使得任意消息从节点 i 传递到节点 j 的延迟不超过\Delta。
2. **节点时钟同步**：所有节点的本地时钟同步，误差不超过\delta。
3. **网络拓扑稳定**：在共识过程中，网络拓扑结构保持相对稳定。

**网络拓扑**：

网络的拓扑结构用邻接矩阵$A = [a_{ij}]_{N \times N}$表示，其中$a_{ij} \in \{0, 1\}$表示节点 i 和节点 j 是否相连。假设网络是连通的，且每个节点的度数至少为$d_{\min}$​​

### 传统比特币用的方式（常被称为“广播”）

- 新区块产生后，**尽量快速、尽量广地**通知全网所有节点
- 实际实现通常是**gossip 协议**（也叫八卦协议、流言传播协议）的变种
- 每个节点收到新区块后，会**转发给它的大部分（甚至全部）邻居**，邻居再继续转发
- 特点：传播很快，但**通信量非常大**（接近每个节点都向很多节点发消息），总通信量接近 O(N²) 量级（N 是节点数）

### 论文（LCLCS）

论文采用的传播策略：

1. **节点平时只跟自己的少数固定邻居（稀疏连接，平均度数 ≈ log N）交流**
2. **不是“有新东西就发”，而是主动周期性地“问邻居”**：你现在链有多长？权重多少？
3. **只有当发现邻居的链明显更好时，才去要整条链**（惰性同步）
4. **判断逻辑是本地最优的**： “如果我比所有邻居都长 → 我暂时觉得自己可能是全局最长的，不主动发” “如果邻居比我长 → 我立刻认输，去抄邻居的”
5. **真正的好链会像滚雪球一样通过这种“问-比-抄”的级联方式慢慢传开**


原先的算法：

**经典最长链共识**（Classic Longest Chain Consensus）是指以比特币为代表的、由中本聪在 2008 年比特币白皮书中首次提出的 Nakamoto 共识机制的核心链选择规则。其基本思想可以用一句话概括：

> 在所有合法链中，节点始终选择并在其上继续工作的链，是**从创世区块开始累计工作量（proof-of-work）证明最多的那一条链**。在比特币的原始设计中，这通常近似等价于**区块数量最多的链**（即最长链）。

1. **链的有效性** 一条链从创世区块开始，每一个区块都必须：
    - 满足工作量证明难度要求
    - 区块头中的前一区块哈希正确指向上一区块
    - 区块内所有交易合法
    - 时间戳合理（比特币有较宽松的时间戳规则）
2. **链的排序标准**（Fork choice rule） 节点在面对多条合法链时，选择的标准是： **累计工作量（Cumulative Proof-of-Work）最大的那条链** 累计工作量 = 该链上所有区块难度之和 （在比特币早期及大多数时期，由于难度调整机制，累计难度与链长度高度相关，因此常被简称为“最长链”）
3. **节点的行为规则**
    - 收到新区块 → 如果该区块接在当前最重链上，则接受并继续在其上挖矿
    - 收到新区块 → 如果该区块接在一条比当前链更重的链上，则切换到那条更重的链（发生链重组/reorg）
    - 如果两条链累计难度相同 → 暂时保留两条链，等待下一个区块决定（先延长哪条哪条获胜）
    - 节点永远只在自己当前认为的最重链上尝试挖矿
4. **安全参数：确认数（confirmation）** 通常认为，一笔交易在最重链上被后续区块确认了 k 个区块后（k=6 是比特币社区最常用的经验值），其被逆转的概率已经极低，可以认为交易“基本确定”。

**低复杂度最长链共识算法（LCLCS）** 的完整流程如下所述。该算法在**同步网络假设**下运行，旨在以远低于传统 PoW 的通信开销实现最长链的收敛。

### 前提假设（必须同时满足）

- 网络是**同步的**：任意两条节点间消息传递延迟存在明确上界 Δ
- 节点本地时钟基本同步（误差远小于 Δ）
- 网络拓扑相对稳定，且为连通图（通常假设稀疏拓扑，平均度数 O(log N)～ O(log log N)）
- 诚实节点占绝大多数（典型地>50%～ 67%，具体安全阈值依赖分析）
- 存在一条客观上累计工作量/长度最长的诚实链

### 算法主要流程（每个节点的视角）

每个节点持续独立执行以下循环（通常以固定时间片或事件驱动方式实现）：

1. **本地新区块产生或接收阶段**
    - 如果本节点是出块者：完成 PoW 后生成新区块，**仅添加到本地链**（此时**不立即广播**）
    - 接收到邻居通过请求发来的新区块或整条链时，进行合法性验证：
        - 区块格式、签名、工作量证明、时间戳、前向哈希等全部验证通过
        - 如果合法，则尝试把新区块接到本地链的合适位置
2. **周期性邻居状态查询**（核心通信步骤） 每个节点以固定频率（通常每几秒～几十秒一次，远低于出块间隔）向其所有**直接邻居**（通常 10 ～几十个）发送轻量查询消息： 查询内容通常只包含：
    - 当前链高度（length）
    - 当前链总权重（weight，可为累计难度、累计分数、时间加权等）
    - 可选：链尖区块哈希（用于快速比对）

    → 这步通信量很小，每个查询+响应通常只有几百字节

3. **本地最长链判断**（完全本地计算，无需额外通信） 节点将**自己当前的链**与所有**刚刚收到的邻居状态**进行比较，按照以下**严格优先级**决定：

```
   比较规则（词典序）：
   1. 链高度（length）更大者优先
   2. 高度相同时，链权重（weight）更大者优先
   3. 高度与权重均相同时，保留当前链（或按哈希/时间戳等次要规则打破平局）
```

判断结果分为三种情况：

| 情况 | 判断结果                   | 后续动作                                 |
| ---- | -------------------------- | ---------------------------------------- |
| A    | 本地链 ≥ 所有邻居          | 保持当前链，继续在本地链上工作           |
| B    | 存在至少一个邻居链明显更优 | 选择其中最优的一条（最高 height+weight） |
| C    | 平局（高度、权重都相同）   | 保守策略：暂时保留本地链                 |

4. **惰性链同步**（仅在需要时发生，通信量最大的部分）
    - 仅当判断为**情况 B**时，才向那个“胜出”的邻居发起**完整链同步请求**
    - 接收到完整链（或缺失的后缀）后：
        - 验证整条链的合法性
        - 如果验证通过，则**切换**本地链为这条更优的链（发生本地重组）
        - 如果验证失败，则标记该邻居为不可信（可选择降低其信任度或断开连接）

5. **继续挖矿/验证循环**
    - 不管是否切换，都在**当前本地认为最优的链**上继续进行：
        - 验证交易
        - 打包新区块（如果是出块轮次）
        - 尝试 PoW 计算

### 整体系统层面的收敛过程（宏观视角）

1. 某个诚实矿工挖出真正的最长/最重新区块
2. 该区块首先被其直接邻居通过查询发现（因为邻居高度落后）
3. 这些邻居在下一次查询判断时会切换到这条新链
4. 新链信息以**级联方式**通过“问—比—抄”的机制向外传播
5. 由于网络是同步的 + 拓扑连通 + 诚实节点占多数，最长链信息最终会在**O(网络直径) × 查询周期 + Δ** 的时间内覆盖几乎所有诚实节点
6. 一旦绝大多数诚实节点切换到同一条链，新产生的区块绝大多数都会接在这条链上 → 形成正反馈 → 其他竞争链快速失去竞争力 → 最终收敛

### 1.将**网络动力学理论**应用于最长链共识算法的设计中，为共识过程提供了数学建模

**同步网络假设**：

1. **消息传递延迟有上界**：存在常数\Delta > 0，使得任意消息从节点 i 传递到节点 j 的延迟不超过\Delta。
2. **节点时钟同步**：所有节点的本地时钟同步，误差不超过\delta。
3. **网络拓扑稳定**：在共识过程中，网络拓扑结构保持相对稳定。

将网络动力学领域的连续时间建模方法应用于最长链共识过程，建立了描述共识演化的数学模型：

- 构建了节点链长度 L_i(t) 的连续时间动力学方程
- 构建了节点链权重 W_i(t) 的连续时间动力学方程

- 把离散的、事件驱动的区块传播与链选择过程，抽象为连续的网络耦合动力学系统
- 为“最长链信息如何通过局部交互向全网传播并趋同”提供了理论描述框架
- 为后续收敛性分析、参数影响分析奠定了数学基础

### 2.在同步网络模型下，改进了最长链共识算法的比较对象和触发时机

在明确同步网络假设（消息延迟有上界、时钟基本同步）的前提下，对传统最长链共识的两个关键决策维度进行了根本性调整：

**① 缩小比较对象范围**
在同步网络假设下，节点不需要与所有节点通信即可确定最长链。通过设计基于本地决策的最长链选择策略，节点仅需与部分邻居通信即可判断最长链。

传统方式：节点需将本地链与已知所有竞争链比较，依赖全网广播收集信息
改进方式：节点仅与当前直接邻居的最新状态（链高度 + 链权重）进行比较，比较范围从全网缩小到稀疏图中的邻居子集

**② 触发时机从事件驱动改为周期驱动**
传统方式：每收到一个新区块或本地产生新区块时立即触发比较
改进方式：节点以固定时间间隔主动向邻居发送轻量查询，收到响应后才进行比较与决策

### 3. 提出了基于链权重的快速收敛机制

设计了多维度链权重计算方法，以加速分叉解决与共识收敛：

区块权重 w(b) = w_work(b) + w_time(b) + w_priority(b)
其中：

- w_work(b)：传统工作量权重（如PoW难度）
- w_time(b)：时间衰减权重，采用指数衰减形式：α · exp(−λ(t − t_b))，使较早区块获得更高权重
- w_priority(b)：可选优先级权重（可根据交易重要性等设定）

建立了加权最长链选择规则，优先选择链长度最大的链，长度相同时，选择总权重最大的链
通过权重耦合项增强动力学模型中权重信息的传播效率

### 1. 将网络动力学理论扩展应用于异步网络环境下的最长链共识，并建立概率性数学建模框架

- 将网络动力学连续时间建模方法首次系统应用于**异步网络**条件下的最长链共识过程。
- 建立了适应异步特性的节点状态演化方程，主要包括：
    - 链长度动力学方程：引入**概率传递项 p\_{ij}(t)** 与随机延迟 τ*{ij}(t) dLᵢ(t)/dt = αᵢ(t) + β ∑ p*{ij}(t) ⋅ max(0, 延迟接收的邻居长度差) + ξᵢ(t)
    - 链权重动力学方程：类似结构，同样纳入概率与延迟因素。
- 通过概率传播机制 p\_{ij}(t) = exp(−λ⋅τ) ⋅ (1−p_loss) 来表征异步网络中消息有效性随延迟和丢失概率的衰减。
- 为异步环境下“最长链信息以概率方式逐步趋同”的过程提供了统一的数学描述框架。

### 2. 设计了基于概率传播的低复杂度最长链选择策略

- 在完全异步网络假设下，重新设计了最长链判断与选择机制，使其适应延迟无界、时钟不同步、消息乱序等极端条件。
- 核心优化仍然集中在两个维度，但与同步版本有本质区别：
    1. **比较对象**：仍限定为**直接邻居**的最近状态（高度+权重），保持 O(log N) 通信规模。
    2. **信息收集与决策方式**：从确定性周期查询改为**时间窗口内概率性收集 + 软概率选择**。
- 引入了**概率性链评估与选择**：
    - 使用 softmax 形式计算每条候选链成为全局最长链的相对概率
    - 节点可以**确定性地选概率最高者**，也可以**按概率随机抽样选择**（增加探索性）
- 通信复杂度理论上仍保持在 O(N log N)（在消息成功率较高时）。

### 3. 提出了基于时间窗口的异步收敛机制（核心工程创新）

- 针对异步网络的根本难题（延迟无界、乱序、丢失），设计了**自适应时间窗口机制**作为异步环境下信息收集与排序的基础设施：
    - 时间窗口大小 Δt(t) 动态根据当前网络延迟估计（均值 + 标准差）自适应调整
    - 节点在滑动时间窗口内收集所有到达的消息（而非固定周期）
    - 利用消息携带的**发送时间戳**对乱序消息进行逻辑重排序
- 实现了对异步网络中消息延迟、乱序、部分丢失的有效工程化处理
- 实验表明，自适应时间窗口相比固定窗口或无窗口机制，在收敛时间和消息顺序错误率上均有显著改善。
